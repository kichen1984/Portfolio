How I built this site
  1. I designed a prototype via Webflow which is a mix of a code and no-code platform that allows for the output of raw HTML, CSS and Javascript.  

    They can host on their platform internally with an inbuilt CMS, or you can host it yourself and apply an external headless CMS if needed. In this case the content will be mainly static so the exclusion of a CMS and no blog section on the site makes taking the code elsewhere desirable.

    I chose a decentralized storage infrastructure to host and deploy the site code. This was to insure maximum uptime, quick local node access to the site content, full censorship resistance and to prove how easy it is to update your websites to web3 infrastructure.

    So the first step here was to design the site in webflow much like you would probably use Figma, add any custom code needed, and then export the final code out of webflow to be hosted on a github private repo ( I will make the repo public soon)
  2. To deploy the sites code and host it's content I wanted to avoid larger centralized services such as AWS or Heroku.

    FLEEK makes this incredibly simple from the compiling and deploying of your production build, to integrating critical DNS features. More importantly it hosts all the content to IPFS to insure censorship resistance. You can also use Saas offerings such as Pinata to host and pin important IPFS content providing the same advantages.
